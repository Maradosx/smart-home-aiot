 บทสรุปปริญญานิพนธ

ชื่อปริญญานิพนธ์

**บ้านอัจฉริยะ AIoT (Smart Home AIoT)** — ระบบควบคุมบ้านอัจฉริยะแบบสองภาษาด้วย AI ที่ตอบสนองตามบริบทของบ้านจริง


 ที่มาและวัตถุประสงค์

ระบบ Smart Home ที่มีอยู่ในตลาดส่วนใหญ่ขาดความสามารถในการ "เข้าใจบริบทของบ้าน" ผู้ใช้ต้องสั่งงานในรูปแบบที่ตายตัว ระบบไม่สามารถตัดสินใจจากสภาพแวดล้อมจริง เช่น เวลา ตำแหน่งผู้ใช้ สถานะอุปกรณ์ หรือสภาพอากาศ ทำให้การใช้งานไม่ลื่นไหลและเสี่ยงต่อการสั่งงานผิดพลาด

โครงงานนี้พัฒนาระบบบ้านอัจฉริยะที่ใช้ **AI ระดับสูง (Anthropic Claude Haiku 4.5)** ในการตีความคำสั่งร่วมกับ **homeContext 7 fields** (GPS, เวลา, สภาพอากาศ, sensors, devices, house_mode, comfort_mode) เพื่อให้ระบบเข้าใจบริบทของบ้านก่อนตัดสินใจ โดยมีวัตถุประสงค์หลัก 4 ข้อ ได้แก่ (1) รองรับการสั่งงานสองภาษา ไทยผ่าน LINE Bot "ชาวี" และอังกฤษผ่าน Amazon Alexa (2) ทำงานอัตโนมัติตาม sensor + GPS + สภาพอากาศ (3) มีระบบความปลอดภัยใช้งานจริง ตรวจจับแก๊สรั่ว น้ำรั่ว การล้ม และผู้บุกรุก และ (4) มีระบบแจ้งเตือนที่ทนทาน 3 ช่องทาง (Mobile + LINE + Dashboard) เพื่อป้องกันข้อมูลตกหล่น

หลักการทำงาน

ระบบใช้สถาปัตยกรรมแบบ 3 ชั้น (3-Layer Architecture) ประกอบด้วย

**ชั้นที่ 1 — Edge Layer** ใช้ไมโครคอนโทรลเลอร์ ESP32 DevKit V1 จำนวน 5 ตัว ติดตั้ง 5 โซน (ห้องนอน ห้องนั่งเล่น ครัว ห้องน้ำ+สวน โรงรถ) เชื่อมต่อกับเซนเซอร์ DHT22, PIR, LDR, MQ-2 (แก๊ส), LD2410 (mmWave fall detect), Capacitive Soil และ Actuator (Relay, Servo SG90) ส่งข้อมูลผ่าน Wi-Fi 2.4GHz ไปยัง MQTT Broker นอกจากนี้ Raspberry Pi 3B+ ทำหน้าที่เป็น Zigbee Coordinator ผ่าน Zigbee2MQTT รองรับอุปกรณ์ Zigbee 6 ตัว (Tuya RGB Bulb, Tuya Smart Plug ×2, Sonoff SNZB Motion/Door/Water Leak)

**ชั้นที่ 2 — Cloud Layer** ใช้ AWS EC2 t3.small (us-east-1) รัน Home Assistant Core 2026.4.2 และ Mosquitto MQTT Broker ผ่าน Docker, AWS Lambda รองรับ Alexa Custom Skill (23 intents), n8n บน Railway รัน workflow Daily Report และ HA Alert, Supabase PostgreSQL บันทึก chat_logs และ mode_changes สำหรับ analytics

**ชั้นที่ 3 — AI Layer** ใช้ Anthropic Claude Haiku 4.5 (claude-haiku-4-5, context 200K tokens) เป็น Shared AI Brain ทั้งสำหรับ LINE Bot ภาษาไทยและ Alexa ภาษาอังกฤษ AI รับ homeContext ทุกครั้งเพื่อใช้ตัดสินใจในรูปแบบ JSON action ระบบมี automation ทั้งหมด 87 รูปแบบ ครอบคลุม Smart Light, Auto Pump, Scene System, Mode System, Safety Response และ Notification Fan-out


สรุปผล

โครงงาน Smart Home AIoT ผ่านการสอบปกป้องในวันที่ 26 พฤษภาคม 2569 โดยได้รับการชื่นชมจากคณะกรรมการในด้าน Engineering Maturity, AI Sophistication และ Real-world Value ผลการทดสอบ AI ทั้ง 3 ด้านบนชุดทดสอบ 1,300 ตัวอย่าง พบว่าระบบมี **ความเร็วการตอบสนองเฉลี่ย 1.6 วินาที**, **ความแม่นยำในการตีความคำสั่ง 96.8%** และ **ความปลอดภัย 98.5%** (GPS Guard + Hallucination Prevention) ซึ่งสูงกว่าเกณฑ์เป้าหมายทุกด้าน

ระบบสามารถใช้งานจริง 24/7 ในบ้านของผู้พัฒนาไม่ใช่เพียง demo สำหรับวันสอบเท่านั้น ค่าใช้จ่ายในการดำเนินการรวมต่อเดือนอยู่ที่ประมาณ 1,664–1,864 บาท ภายในงบประมาณที่ตั้งไว้ โครงงานนี้แสดงให้เห็นถึงความเป็นไปได้ของ **Context-Aware Smart Home** ที่ใช้ AI ระดับสูงในการตีความและตัดสินใจ ซึ่งเป็นจุดเด่นที่แตกต่างจาก commercial smart home ทั่วไป และสามารถต่อยอดในเชิงพาณิชย์หรือนำไปประยุกต์ใช้กับผู้สูงอายุที่อยู่บ้านลำพังได้
